# Eval Checklist

## 通用验收

每个阶段结束前必须检查：

- 是否遵守当前阶段范围。
- 是否没有引入未批准的业务边界漂移。
- 是否更新了 `state/current-state.md`。
- 是否有明确测试或验证记录。
- 是否记录新增债务。
- 是否提供 handoff。

## Harness 阶段验收

- `docs/harness` 文件清单完整。
- `01-current-architecture.md` 明确标注“文档阶段 / 未落地代码”。
- 未创建业务代码。
- 未创建 Spring、Vue、React、FastAPI 项目。
- Phase backlog 从 MVP 开始，并包含用户指定的 7 个优先阶段。

## 契约验收

- OpenAPI 或事件 schema 有版本。
- 请求、响应和错误示例齐全。
- 状态枚举有生命周期说明。
- 租户上下文、权限、审计字段明确。
- 破坏性变更有迁移说明。

## 后端验收

- 服务只拥有自身边界内的数据和行为。
- 数据库迁移可重复运行。
- 租户隔离被测试覆盖。
- 关键接口有单元测试和集成测试。
- 错误码和响应格式遵循契约。
- 业务动作有审计记录。

## Agent 验收

- LangGraph 或等价状态机有清晰状态。
- 每个关键步骤有输入、输出、状态和耗时记录。
- 工具调用遵循契约。
- 高风险动作不直接执行。
- 失败有可追踪错误码。
- 报告中的结论区分事实、推断和建议。

## RAG 验收

- 文档有解析状态。
- 切片可以追溯到原文。
- 检索结果带来源。
- 报告引用可追踪。
- 解析失败可重试。
- 至少有最小评测集覆盖命中率和引用准确率。

## 前端验收

- 关键流程有加载、空状态、错误状态。
- 页面通过 API 契约消费数据，不依赖内部表结构。
- Agent 执行链路可读。
- 报告引用和风险标记可见。
- 租户和权限错误有明确提示。

## 安全验收

- 所有业务数据按租户隔离。
- 敏感字段不在默认视图明文展示。
- Token、密钥和模型凭据不进入仓库。
- 高风险工具需要审批或策略阻断。
- 模型输入、输出和工具调用可审计。

## AI 质量验收

MVP 最低指标先定义为可测，不强行设高阈值：

| 指标 | 含义 | MVP 验收 |
|---|---|---|
| RAG 命中率 | 检索结果包含目标答案来源的比例 | 有样例集和计算方式 |
| 引用准确率 | 报告引用能追溯到真实片段的比例 | 有样例集和计算方式 |
| 幻觉率 | 无来源或错误结论比例 | 有人工标注入口 |
| 工具调用成功率 | Agent 工具调用成功比例 | 有日志和统计 |
| 工作流完成率 | 任务从发起到报告完成比例 | 有状态统计 |
| 人工介入率 | 低置信度或高风险转人工比例 | 有状态统计 |

## 阻断条件

出现以下任一情况不得进入下一阶段：

- 业务代码绕过 `shared-contracts` 私自定义跨服务字段。
- Agent 可直接执行高风险业务动作。
- 当前架构文档与已落地代码明显不符。
- 租户隔离缺失。
- 无法复现最小验证流程。
- 未记录已知严重债务。
